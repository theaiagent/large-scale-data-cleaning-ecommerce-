Cleaning Actions Performed

Each cleaning step was applied programmatically. Row counts are computed from direct comparison of the raw and cleaned datasets.

Issue Rows Affected Action Taken
Duplicate Rows 5,000 Identified and removed exact duplicate records to ensure each transaction appears once.
UTF-8 Encoding Corruption 37,978 Repaired Mojibake artifacts (e.g. “Ã©” → “é”) by detecting and reversing double-encoded UTF-8 sequences.
Currency Symbols in Prices 74,901 Stripped currency prefixes/suffixes (€, $, £, TL) and normalized European comma decimals to produce clean float values.
Inconsistent Date Formats 99,933 Parsed 4+ date formats (DD.MM.YYYY, Mon DD YYYY, MM/DD/YYYY, DD-MM-YYYY) into ISO 8601 (YYYY-MM-DD).
Status Typos & Casing 62,294 Corrected misspellings (“Cancellled”, “deliverred”) and normalized all statuses to lowercase (5 canonical values).
Country Name Inconsistencies 73,483 Mapped full names and variants (“germany”, “United States”, “UK”) to ISO 3166-1 alpha-2 codes.
SKU Format Variations 31,147 Standardized mixed formats (SKU003, sku-002, Sku-010) to uniform uppercase with dash (SKU-003).
Unused / Empty Columns 3 columns Dropped 3 columns (_unnamed_1, _unnamed_2, notes) that were entirely null or unused.
Placeholder Strings as Nulls Verified Converted sentinel strings (N/A, n/a) to proper null values for consistent missing-data handling.
Phone Number Formatting 81,346 Standardized varied phone formats (555.819.8173, +1-555-6983, 5554475) to a consistent pattern (555-XXX-XXXX).

The cleaning pipeline is modular and reusable for future dataset updates. Each step can be independently configured or extended for different data sources. The approach prioritizes reproducibility — running the same script on the same input will always produce identical output.
Approach Highlights
  • Reproducible Pipeline — Deterministic transformations documented as code, producing identical results on every run
  • Automated Validation — Built-in assertions verify data integrity after each cleaning step
  • Modular Architecture — Each cleaning function handles one concern and can be toggled or reconfigured independently
  • Graceful Error Handling — Malformed records are flagged and logged rather than silently dropped
  • Scalable Design — Tested on 125K rows; architecture supports datasets of 1M+ rows with minimal changes
